## Implemented Scope

Mining targets now live in `mining_data.py` and are exported through `constants.ORE_DATA` using stable target IDs. Inventory remains keyed by real item names.

Implemented source rows:

- Rune essence / Pure essence
- Clay
- Copper
- Tin
- Blurite
- Iron
- Silver
- Coal
- Gold
- Mithril
- Adamantite
- Runite

Implemented scoped simple historical rows:

- Limestone
- Sandstone, as one weighted-output target
- Granite, as one weighted-output target
- Gem rocks, as one weighted-output target

Deferred:

- Concentrated coal/gold. These are mostly depletion/location distinctions in this review loop unless Living Rock Caverns behavior is modeled.
- Shooting Stars. This needs a separate activity/timing/reward design.
- Mining random events. The old pickaxe-head/rock-golem style events do not fit the target era or the review reward loop.

## Mechanics Decisions

- Pickaxes use the same hybrid model as Woodcutting: bound toolbelt IDs plus owned real pickaxe items.
- Existing saves receive `toolbelt["mining"] = ["bronze_pickaxe"]`.
- Source low/high roll values and pickaxe cadence are adapted to one-review-attempt probabilities in `logic_pure.py`; source ticks are not simulated directly.
- Random incidental gems are side drops and do not grant Mining XP.
- Amulet of glory uses the source-shaped boosted gem chance.
- Varrock armour bonus is backend-only through `owned_equipment`; it grants an extra output item without adding extra XP, matching the local plugin's reward flow.

## Tradability Decisions

The item manifest now carries explicit `tradeable` metadata.

Source-aligned entries:

- Blurite ore: non-tradable.
- Inferno adze: non-tradable and deferred.
- Normal pickaxes: tradable.
- Gilded pickaxes: tradable.
- Incidental/gem-rock uncut gems: tradable item forms, not noted forms.
- Varrock armour 1-3: non-tradable.

Normalized anomalies:

- Dragon pickaxe: local `PickaxeType.kt` points at item `15259`, but `items.yml` describes `15259` as a non-tradable bad item with an unrelated examine string. Item `15261` is the tradeable "Used for mining" Dragon pickaxe. AnkiScape uses `15261`.
- Varrock armour 4: local `items.yml` marks `19757` as tradeable, unlike diary reward equipment. AnkiScape normalizes it to non-tradable with a note in `mining_data.py`.
