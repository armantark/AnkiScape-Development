# Product Context

## Why This Exists
Anki already rewards consistency through memory gains, but the feedback loop can feel abstract. AnkiScape adds small visible progression so each review can feel like moving a character forward.

## User Experience Principles
- The study session is primary; game feedback must be quick, legible, and optional.
- The user should be able to pick a skill, review cards normally, and see progress without managing a second app.
- Rewards should feel persistent enough to motivate long streaks, but not so complex that they distract from answering cards.
- Progression should support long-term grind without requiring all mechanics to be present at once.

## Current Experience
- A review HUD shows the active skill, level, and progress.
- Floating XP appears after earned XP.
- Level-up and achievement dialogs can appear when milestones are met.
- A floating widget and main menu expose skill selection, item selection, bank, stats, achievements, and settings.
- Four skills exist now: Mining, Woodcutting, Smithing, and Crafting.

## Expansion Direction
The desired direction is "OSRS-like breadth inside Anki": eventually many skills, item chains, and possibly combat. The first expansion should still feel like AnkiScape.

## Melvor Idle Boundary
There is conceptual overlap with Melvor Idle because both sit near idle-RPG skilling. Use that overlap to validate that progression, item chains, and passive-feeling loops can be fun. Do not treat Melvor as a spec, content source, balancing source, or UI template. Consider how they handle combat from their async-ish loop.
