# Equipment Combat-Ingestion Contract

## Status

Backend and frontend complete as of 2026-06-02. The Equipment tab, right-click context menus, tooltips, slot icons, and stat totals are wired to the backend contract.

## Contract

`equipment_data.py` is generated by `tools/generate_equipment_data.py` from:

- checked-in `smithing_data.py` recipe metadata,
- local 2011Scape `data/cfg/items.yml` equipment blocks,
- `mining_data.py` Mining bonus items.

Runtime imports only the checked-in generated Python data. The 2011Scape checkout remains a developer-time audit input.

## Save Shape

`player_data["equipment"]` is the single worn-equipment source of truth:

```python
{"weapon": "Bronze sword", "body": "Bronze platebody"}
```

Slots are the stable 11-slot surface in `EQUIPMENT_SLOTS`: head, cape, neck, ammunition, weapon, body, shield, legs, hands, feet, ring. `owned_equipment` is removed during migration.

Combat skills are scaffolded as flat save keys. Planned combat levels default to 1, except Constitution at 10. Combat XP defaults to 0. The combat skills remain untrainable until their real systems land.

## Equipment Data

Each `EquipmentItem` exposes:

- `slot`
- `combat_skill`
- `required_level`
- `two_handed`
- `equipment_type`
- `bonuses`
- `attack_speed`
- `source_item_id`
- `source`
- `source_status`

`EquipmentBonuses` stores the 2011Scape stat block: attack stab/slash/crush/magic/ranged, defence stab/slash/crush/magic/ranged, melee strength, ranged strength, magic damage, and prayer.

## Pure Mechanics

`logic_pure.py` owns the backend rules:

- `can_equip_item_pure` gates by `player_data[f"{combat_skill}_level"] >= required_level`.
- `equip_item_pure` moves one inventory item into its slot, returns displaced gear to inventory, and enforces two-handed weapon vs shield displacement.
- `unequip_item_pure` returns a worn item to inventory.
- `equipment_stat_totals_pure` sums combat bonuses across worn slots.
- `equipment_bonus_state_pure` replaces the old passive `owned_equipment` Mining bonus check. Amulet of glory only boosts gem chance when worn in `neck`; Varrock armour only grants extra ore when worn in `body`.

## Frontend Handoff

Frontend should treat the backend as ready when it needs:

- `constants.EQUIPMENT_DATA`
- `constants.EQUIPMENT_SLOTS`
- `logic_pure.can_equip_item_pure`
- `logic_pure.equipment_stat_totals_pure`
- runtime `on_equip_item(item_name)`
- runtime `on_unequip_slot(slot)`

The Bank tab now stays focused on inventory plus toolbelt. Worn gear lives in the dedicated Equipment tab, which calls the runtime callbacks above and refreshes in place.

## Manual Test Instructions

Manual runtime verification should happen inside Anki when equipment behavior changes:

1. Seed inventory with `Bronze platebody`, `Rune platebody`, `Bronze 2h sword`, and `Bronze sq shield`.
2. Confirm Bronze gear equips at default combat levels.
3. Confirm Rune gear is disabled or rejected with "Requires level 40 Defense".
4. Equip a shield, then equip a 2h sword and confirm the shield returns to inventory.
5. Equip a shield while a 2h sword is worn and confirm the weapon returns to inventory.
6. Equip Amulet of glory and Varrock armour, mine Coal, and confirm Mining bonuses are sourced only from worn slots.
