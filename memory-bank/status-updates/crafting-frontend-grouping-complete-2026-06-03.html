What shipped

1
Family-grouped tree + expand persistence _build_craft_list

Grouped by CraftingRecipe.family, collapsed by default, current target's family auto-expands, state round-trips through ankiscape_craft_expanded_families. Stable recipe ID on UserRole, persisted via on_set_craft. Tooltips: output qty, base XP, per-material owned counts.

2
Bug fix: locked rows were selectable all 6 builders

Qt fires itemClicked even on disabled rows (the enabled flag gates selection, not the click), so clicking a greyed target made it active → next review failed with "you don't have X". Every handler (ore, tree, smith, craft, fletch, utility) now ignores disabled rows.

item.isDisabled() / flags() & ItemIsEnabled
3
Full crafting icon set tools/fetch_crafting_assets.py

Manifest derived from CRAFTING_DATA (every output + crafting-only raw materials; ores/gems/bars/logs skipped — registry resolves those elsewhere). 74 fetched, 0 failed, alpha-trimmed. constants now wires CRAFTED_ITEM_IMAGES for outputs and input materials so the Bank shows them too.

Skill MD hardened ankiscape-skill-expansion

The live-content non-negotiable now forbids any tag/label/badge/state for emergently-inaccessible content, requires the disabled-row click guard, and makes asset fetching mandatory every pass — so this doesn't need re-specifying.

Tests + click guard, + icon coverage

Qt: dependency-heavy targets present & untagged & material-gated; disabled-row click no-ops on Crafting and Smithing trees. Registry: every Crafting output resolves an icon, key raw materials wired. Reverted the v1 input-starved data test.
