Thread Boundary

Backend owns now

Data generation, save migration, pure equip/unequip rules, stat totals, runtime callbacks, and backend tests.

Frontend waits

Dedicated Equipment tab, row context menus, slot placeholder icons, compact bonus tooltips, and Qt behavior tests.

Frontend handoff is ready once the stable APIs exist: `EQUIPMENT_DATA`, `EQUIPMENT_SLOTS`, `can_equip_item_pure`, `equip_item_pure`, `unequip_item_pure`, `equipment_stat_totals_pure`, and the two runtime callbacks.
