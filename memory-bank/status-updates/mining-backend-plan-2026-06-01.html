Pre-Implementation · Backend First

Mining gets source-shaped weight.

This pass takes Mining from display-name ores and generic odds to Woodcutting-style backend parity: stable target IDs, pickaxe-aware rolls, source-modeled gems, safe migrations, and a tiny backend-only equipment foundation for Mining bonuses.

Backend Route

Five contracts before the frontend agent touches Qt

1
AuditLocal rev 667 plugin first, then ordinary missing rocks like sandstone, granite, limestone, and gem rocks if source-clean.
2
TargetsStable Mining IDs in `ORE_DATA`; inventory remains real item names such as `Coal` and `Pure essence`.
3
ToolsToolbelt remains tools-only, with starter bronze pickaxe and source-normalized tiers.
4
BonusesMinimal owned-equipment backend collection for Glory and Varrock armour effects, no acquisition/UI yet.
5
ReviewOne eligible successful card is one attempt; source-shaped RNG, undo-safe mutation, XP multiplier after base XP.
