Provenance none

No smithing_data.py, no source audit doc. Everything is inherited legacy data.

This pass creates both, matching the Mining/Woodcutting structure.

The loop this closes (why forging matters here)

Mining ore→ Smelt bar→ Forge item→ Pickaxe / Hatchet↺ Mining / Woodcutting toolbelt

Smithed pickaxes & hatchets are real tool item-names the existing toolbelt logic already recognizes (best_mining_pickaxe_pure / best_woodcutting_axe_pure scan inventory by display name), so forging a Rune pickaxe immediately upgrades your gathering. Dart tips & arrow tips feed Fletching; bolts(unf) already exist in Crafting. That makes a tools+ammo forge slice a genuinely closed chain, not dead-end combat gear.

Proposed split (after you pick scope)

Backend — GPT thread

  1. Source audit doc memory-bank/source-audits/smithing-2011scape-*.md.
  2. smithing_data.py: stable IDs for bars + forge recipes, levels, XP (smelt table + bars×barXP), inputs/outputs, produced counts, source notes; legacy BAR_DATA mapping.
  3. Pure mechanics in logic_pure.py: smelt + forge, hammer requirement, one-card = one item (or batch), XP after global multiplier; undo-safe.
  4. item_registry.py + storage defaults/migration; achievements; backend tests.

Frontend — Claude thread

  1. Split the Smithing panel into Smelt vs Forge target lists (or a tier filter), mirroring the registry-driven row pattern.
  2. Rows: display name + level; tooltip with inputs, output qty, base XP, lock reasons (level / missing bars / no hammer).
  3. Stats/Bank surfaces for new forged items; offscreen Qt tests.

Assets pass: extend tools/fetch_mining_assets.py sibling → fetch_smithing_assets.py for the chosen item set + provenance.

Open balance notes

· Iron 50% smelt-fail (SmeltingAction.rollIronBar, interpolates 50→80% over lvl 15–45) and the Ring of Forging are source mechanics we will document but not model — consistent with how Crafting/Mining treat one successful card as one finished unit (no fail rolls in the review loop).

· Batching: smelting and ammo (dart/arrow tips, bolts, nails) are bulk actions in-game; candidates for up-to-28 batch per card like spinning. Single weapons/armour = one per card. Decide per-target in the data.

· Audited base XP stays in data; the global ankiscape_xp_multiplier applies at reward time only.
