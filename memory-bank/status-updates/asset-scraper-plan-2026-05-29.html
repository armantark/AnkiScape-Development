Pre-Implementation · Tooling Only

One icon at a time, with receipts.

This pass adds a standalone asset-fetch CLI that is never imported by AnkiScape runtime. It resolves a single wiki file, downloads safely, normalizes if requested, and records provenance for future cleanup.

Fetch Path

OSRS first, RS3 fallback

1
Name mapping

Storage keys and display names resolve to wiki file titles, with explicit overrides when needed.

2
MediaWiki API

Query imageinfo metadata with a descriptive User-Agent and retry/backoff.

3
Safe write

Skip existing files unless forced; dry-run prints the source and target without touching assets.

4
Provenance

Record wiki, file page, image URL, sha1, timestamp, and license note per output path.

CLI Shape

Developer tool, not addon code

uv run tools/fetch_assets.py
$ uv run tools/fetch_assets.py "Magic logs" --out trees/Magic.png --size 32 --dry-run
wiki oldschool.runescape.wiki
file File:Magic logs.png
